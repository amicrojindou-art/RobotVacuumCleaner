## [2026-07-15] 阳角"胆小浅弧"修复：包边虚拟贴边分（真值顶点）+ 阳角速成课程 + 凸块课程推迟

### 背景 / 现象

移除激光通道注入后重训的 `vacuum_wf0715_1` 仍然阳角失败（用户家居回放
确认：遇阳角向外偏移、脱离沿墙轨迹）；且总回报远低于早期版本（用户以
0708 曲线对比）。

### 定量诊断

1. **回报低是果不是因**：0715_1 每步收益 ~0.20+（与 0713_1 相当），但
   回合平均 ~350 步就死于丢墙超时 —— 阳角失败 -> 4s 丢墙终止 -> 回合短
   -> 总回报塌。与 0708 的绝对值对比无意义（奖励函数已多次改版：帐篷
   速度分、approach/wobble/press 等安全罚项都是后加的，每步上限从 ~0.38
   降到 ~0.27 是设计使然）；
2. **失败模式已从"背离"变为"胆小"**：0715_1 丢墙后 82% 在右转（方向
   正确），但 2s 只转 -13°（0713_1 为 -54°）—— 浅弧永远够不到 10cm
   量程的侧激光，说明不是反射错误而是【不敢转】；
3. **毒源定位**：对比全部实验，凡带凸块课程的从头重训（0714/0714_1/
   0715/0715_1）阳角全废（<10/35），唯一不带的 0713_1 正常（30/39）——
   凸块回合里"果断右转贴回墙"会随机撞上盲区凸块吃罚，把"果断贴回"
   泛化成危险动作；
4. **奖励结构缺口**：包边圆弧的"直行越角 -> 开始右转 -> 射线扫回墙角"
   过渡段 track=0（侧激光无回波），紧贴转弯全程无直接收入；慢包边
   （>1.2s 宽限）的 ~1.5rad/s 转向还会掉进 seek 防打转罚（-0.24/步）。
   0713_1 能学会属于幸运收敛，课程/毒化稍变就学不出来。
   （几何注：正切包边时侧激光其实会"锁住墙角棱边"续报 ~1cm —— 真正
   无回波的只是转向过渡段，虚拟分补的就是这一段。）

### 改动内容（`tasks/wall_follow_task.py`）

1. **阳角虚拟贴边分**（照搬缺口桥接的成功做法）：课程生成时记录全部
   阳角顶点真值（普通拐角负角的折线交点 + U 型墙端的两个端点角），
   侧激光丢墙且在顶点 0.45m（`WRAP_CORNER_RANGE`）内时，按贴墙同一
   公式对"机身中心到顶点距离 = 半径+1cm"发 track（`wrap_err`），
   progress 门控同式、丢墙罚 ×0.3 —— 紧贴包边圆弧与贴直墙收入连续，
   "紧贴转弯"首次有直接正梯度，胆小浅弧/直行飘走拿不到；包边中转向
   免罚（含超 1.2s 的慢包边；需已捕获过墙，防全盲阶段蹭豁免）。
   家居回放无真值、分支自动关闭（同缺口桥接）；
2. **阳角速成课程 corner drill**（30% 回合，用户建议的"初期加入阳角
   学习"）：首段改短墙（box05/06，0.9m）且强制第一个过渡为拐角、70%
   为阳角 —— 出生 ~2s 即进入包边练习，从第 0 迭代开始密集采样；
3. **凸块课程推迟 + press 门控**：`prot_on` frac>0.2 -> **frac>0.6**
   （先固化包边再学碰撞处置）；press 增加 `side.hit` 门控 —— 只罚
   "贴边巡航中顶盲区凸块"，包边中蹭角（侧激光已丢）不吃 press，
   包边期不再有任何额外惩罚。

### 验证（16 项断言全过 + 训练冒烟）

1. 包边圆弧过渡段（欠转 20°、侧激光真实无回波）：wrap_err=0.002、
   track=0.214 发放，转向免罚、丢墙罚 -0.015（×0.3）；宽弧（0.30m）
   track=0；圆弧上驻留 track=0（运动门控不给白拿）；半径外回退旧包边
   窗口逻辑；
2. press：贴墙顶凸块 -0.010 生效；同一接触但侧激光丢墙 -> 0（不误伤包边）；
3. 课程统计（各 300 次 reset）：drill 87/300≈29% 且第 0 迭代生效；
   阳角真值回合 208~242/300；凸块 frac<=0.6 恒 0、frac≈0.93 时 70/300；
4. 家居回放 wrap_corners 恒空、wrap_err 恒 None；随机墙/家居随机步
   奖励观测全有限；端到端训练冒烟 2 迭代正常。

### 兼容性影响

- 观测/动作维度不变（34/2），奖励 dict 键不变（press 已于上一条加入）；
  **必须重训**，`vacuum_wf0715_1` 作废；
- 预期：回合长恢复 500+，eval 回报回到 100~170 区间（注意与 0708 的
  ~200 无可比性，奖励函数不同）；家居阳角（立柱/家具角/门洞侧柱）
  按课程泛化包边；
- 若重训后包边仍偏胆小，下一步调参顺序：CORNER_DRILL_PROB 上调 /
  PROT_PROB 下调或 prot_on 更晚 / WRAP_CORNER_RANGE 微调 —— 全部在
  奖励/课程层，不碰观测。

## [2026-07-15] 阳角包边崩坏定案：彻底移除激光通道碰撞注入（虚拟墙/虚拟回波方案废弃），碰撞改纯奖励层处理

### 背景 / 现象

按"±65° 触发锥 + 虚拟墙记忆"重训的 `vacuum_wf0715` 仍然失败（用户回放
确认）：沿墙遇到立柱等阳角障碍时向外偏移、脱离沿墙轨迹，不能紧贴转角
包边；且各版本评估回报越来越低。

### 定量诊断（headless 回放，逐步记录）

1. **回报低 = 回合早死，不是每步变差**：0715 每步收益 ~0.27（0713_1 为
   ~0.25），但回合平均只活 ~313 步就死（0713_1 后期 ~530）；随机墙 40 回合
   死因 36 个 `lost_wall` —— 全是丢墙飘走；
2. **阳角包边在训练分布内就崩了**：0715 普通（无凸块）回合阳角事件
   27 个、2s 内贴回 **0 个**；
3. **对照定位到环境层**（冻结 0713_1 策略、同一环境）：注入开启时包边
   24/40、回合长 431；**运行时关掉注入 -> 恢复 30/39、回合长 508~561**，
   失败样本 100% 在右转朝墙（只是慢）。
   ==> 崩坏由观测注入本身造成，与策略、与触发锥宽窄无关。

### 根因（原理性，已三次重训验证，禁止再犯）

**"紧贴阳角包边"与"碰撞 = 正面有墙"在原理上不相容**：紧贴包边必然偶尔
蹭到墙角，而蹭角接触点在机体系里的方位角会**随右转包边从 -90° 向 0°
自然迁移**，任何宽度的触发锥都会被扫进去；一旦触发，观测变成"正前 1cm
有墙"，训练成熟的 head_on/内角反射（刹停+左转）恰好把机器人推离墙 ——
包边动作被系统性打断，训练中策略进而学会"根本不包边"。0714（wobble
豁免）/0714_1（±90° 注入）/0715（±65° 锥+虚拟墙）三次失败同根。

### 改动内容（碰撞信息只走"接触方位观测 + 奖励项"，激光通道保持纯净）

| 文件 | 改动 |
|---|---|
| `tasks/wall_follow_task.py` | ① **删除虚拟墙/虚拟回波全部机制**（`bump_wall`/`_bump_wall_range`/`BUMP_ECHO_DIST`/`BUMP_WALL_TTL`），前向激光回归纯读数；② 新增 **press 顶推罚**（0.12）：正面锥（±65°，`_chassis_contact` 保留该判定但只喂奖励）内接触且仍在前进 -> 罚前向速度 —— 盲区碰撞的"刹车老师"，侧蹭/包边蹭角（≈±90°）与倒车脱离不罚；③ **接触期 wobble 打 3 折**（`CONTACT_WOB_RELIEF=0.7`）：撞上后转身脱离的必要转向不再被全额压制（0713_1 来回碰撞根因之一），折扣上限 0.175 < scrape 0.20，"故意碰墙换转向自由"必亏（0714 全额豁免的教训）；④ 凸块课程密度 `PROT_PROB` 0.25->0.15（必撞回合太密会带坏碰撞卫生）；⑤ 文件头写入"重要教训"防再犯 |
| `envs/vacuum/vacuum_wf_env.py` | 观测前向通道回归纯激光读数（删除注入分支），文档同步 |

### 验证

1. **14 项断言全过**（RL312）：正面盲区碰撞时奖励层/观测层前向通道均无
   注入（obs[14]=1.0/obs[15]=0）、press 碰撞瞬间生效 -0.052、approach 不再
   被碰撞触发；侧蹭不算顶推；接触期 wobble -0.139 -> -0.009（转向可行）
   且省额 0.129 < scrape 0.20（不可刷分）；无接触贴墙转向严罚不回归；
   家居贴墙出生正常；随机墙/家居随机步全有限；
2. **行为恢复（冻结 0713_1 新环境回放）**：普通回合阳角 2s 内贴回 0/27
   -> **30/39**，未贴回 24 -> 2，回合长 431 -> 508；
3. 端到端训练冒烟 2 迭代正常（本机 RL 环境，见上一条的 RL312/ray 说明）。

### 兼容性影响

- 观测/动作维度不变（34/2）；奖励 dict **新增 `press` 键**（PPO 用
  sum(values) 不受影响，回放脚本按 keys 泛化打印不受影响）；
- **必须重训**；`vacuum_wf0714/0714_1/0715` 三个模型全部作废；
- 预期重训后恢复 0713_1 的阳角/内角/缺口/沿边水平；浅凸起处的改善
  （相对 0713_1 的来回碰撞）由 press+转向减税+接触方位观测承担 ——
  若仍不理想，后续只能在【奖励/课程层】继续调，不得再碰激光通道。

## [2026-07-14] 0714_1 全面回退修复：保险杠触发锥收窄 + 虚拟墙记忆；家居出生改贴墙（降难度）

### 背景 / 现象

按虚拟回波方案重训的 `vacuum_wf0714_1` 行为再次恶化（用户回放确认）：

1. **阳角处直接离开墙面**，不再沿边；
2. **普通（随机墙）场景根本不沿边**，不再贴 1cm；
3. 评估数据证实全面回退：回报 60~120 / 回合长 288~423，
   对照 0713_1 的 103~174 / 514~600（回合被提前终止）。

### 根因：虚拟回波的触发条件是"整个前半圆（±90°）"

`_chassis_contact` 用 `机体系 lon > 0` 判"正面碰撞"——贴边沿墙的**常态
接触恰好都落在这个区域**：

- **贴边侧蹭（方位角≈-90°~-60°）**：1cm 贴边巡航的偶发右侧轻蹭被翻译成
  "正前 1cm 有墙、置信 1"→ approach 限速瞬间归零，巡航速度 0.28 全额吃
  -0.22/步 —— 训练里贴得越紧蹭得越多，策略学到"贴墙危险"→ 不再沿边；
- **阳角包边蹭角（≈-90°）**：包边右转中右前肩碰到墙角 → 观测变"正面有
  墙"→ 触发 head_on 反射（刹停+**左转**）→ 转身背离墙 → 阳角直接离开。

次要缺陷：纯"接触期注入"在机身弹开瞬间回波就消失；真实内角的前向回波
要持续到转过 ~48°（读数 0.185/cosθ−0.175 超量程才消失）—— 反射链走不完，
来回碰撞隐患仍在。

### 改动内容

| 文件 | 改动 |
|---|---|
| `tasks/wall_follow_task.py` | ① 触发锥收窄：接触方位角在**正前 ±65°**（`BUMP_CONE_HALF_DEG`）内才算保险杠碰撞（凸块首触：12cm 凸出≈-22°、6cm≈-46°、3cm≈-62°，都在锥内；平墙侧蹭/包边蹭角≈-90° 被排除）；② 虚拟回波改**虚拟墙记忆**（`bump_wall` 世界系平面 + `_bump_wall_range` 射线求交）：碰撞时在正前 1cm 记面墙、持续接触每步刷新，脱离后读数随转向/后退按真实射线几何演化（左转按 cos 律增长、~48° 超量程自然消失、越过平面/转过 90° 失效、TTL 2s 兜底）—— 与真实内角观测分布完全一致；③ 家居出生点改**贴墙出生**（用户要求降难度）：西墙边 x=-1.0±d0（d0∈[1,3]cm）、y∈[0.20,0.45]（基站南侧）、朝 -y 沿墙 ±0.08rad —— 出生即沿边，不再找墙 |
| `envs/vacuum/vacuum_wf_env.py` | 观测前向通道改用 `task.bump_echo`（与奖励层同一融合值，取更近者），历史帧同步 |
| `run_experiment.py` | import 顺序改为 torch 先于 ray（RL312/Windows 下先 import ray 会让 torch 的 c10.dll 初始化失败 WinError 1114）。注：ray 2.40 的 worker 子进程在本机仍无法加载 torch/mujoco DLL —— 本机端到端训练冒烟需用 RL(py3.7) 环境，正式训练在 Linux 服务器不受影响 |

### 验证（24 项断言全过，RL312 环境）

1. 正面盲区碰撞（3cm 矮墙）：触发锥内、虚拟墙建立、奖励层/观测层读数
   1cm/置信 1（obs[14]=0.10、obs[15]=1.0）、碰撞瞬间 approach -0.096、
   碰撞中 wobble=0；
2. 虚拟墙几何：后退 3cm 读数 0.040（=真值）、左转 30° 读数 0.0733

3. 常态侧蹭（沿墙 ±20° 内擦墙）**不触发**（0714_1 根因封堵）；恒力矩

4. 无接触贴墙转向 wobble -0.046（零摇摆罚不回归）；
5. 家居贴墙出生 5/5：位姿正确、右侧激光命中 1~3cm、出生即 acquired、
   静置 60 步漂移 0.1mm；
6. 随机墙 200 随机步 + 家居 100 随机步奖励/观测全有限；端到端训练
   冒烟 2 迭代正常。

- 观测/动作维度不变（34/2），奖励 dict 键不变；前向通道在碰撞后的语义

## [2026-07-14] 浅凸起方案重做：碰撞 -> 前向激光 1cm 虚拟回波（回退 0714 的 wobble 豁免）

### 背景 / 现象

按 07-13 "接触方位 + 碰撞转向豁免 + 浅凸块课程" 三件套重训的
`vacuum_wf0714`，**行为反而恶化**：

1. 家居回放 5/5 回合死于 `scraping` —— 机器人撞上隔断/围墙后**顶着墙
   持续推压**，1.5s 被终止（用户看到的"找到墙后侧激光没对准就重开"）；
2. 凸块测试：6/12cm 凸块处 fwd 变号 47/41 次、超时不得脱身
   （0713_1 同测试反而 0/8 次变号、3.6/6.5s 绕过）；
3. 头对头对齐虽 12/12 成功，但蹭墙步数 2.6 倍于 0713_1（29 vs 11）。

### 根因复盘（0714 三件套错在哪）

1. **"碰撞期+0.6s 豁免 wobble" 把接触常态化**：豁免 + 25%~35% 课程必撞，
   策略学到"接触无所谓、接触时转向自由"，碰撞卫生全面崩坏（顶墙推压、
   对齐蹭墙翻倍）；
2. **压墙的真正约束项是哑的**：教刹车的 approach（权重 0.22）由前向激光
   触发，盲区碰撞时前向激光无回波 —— **顶推只吃 0.2 的 scrape，"刹车
   老师"不上课**；
3. **绕行动作没有正向梯度**：绕凸块时侧激光仍读原墙（track≈1），豁免窗
   一过零摇摆罚（-0.25/步）就砸在转弯半途上，绕行全程净负收益 —— 策略
   退化成来回蹭。

### 新方案：把碰撞翻译成"前方 1cm 突然出现墙"

盲区凸起撞上时，几何上就是一个内角；策略**已经会**内角/head_on 反射
（刹停 -> 原地左转 -> 侧激光贴上新面 -> 沿边），触发器是"前向激光近距
回波"。于是：**机身前半接触 -> 前向激光注入 distance=FRONT_STOP_GAP(1cm)、
confidence=1 的虚拟回波**（真机保险杠 = 0 距离前障的语义），观测与奖励
同步生效：

- 观测里前向通道变成"1cm 有墙"，历史帧同步 —— 碰撞状态落回训练分布，
  复用成熟反射，不发明新技能；

## [2026-07-13] 缺口"前后往复"+ 内角"停死"修复：贴边分改为"在干活才发"

### 背景 / 现象

`vacuum_wf0713`（镜像重训版）回放暴露同族两个行为缺陷：

1. **缺口处前后往复**：家居场景在隔断缺口处、随机墙场景在训练缺口处，
   机器人一直前后移动、无法继续沿边；
2. **阴角（内角）直接停死**：沿墙走到内角、刹停之后不转向，永久停在
   角落里不动。

### 定量诊断（各 40 回合无渲染回放，逐步记录激光/动作/奖励分支）

**缺口**：18 回合含缺口、12 回合到达，其中 **6 回合困死在缺口边缘的往复
极限环**（fwd_vel 变号 50+ 次、驻留 15~17s 直到回合耗尽），仅 5 回合穿越。
逐步日志（ep03，缺口宽 0.32m）：

- 贴墙推进中 `track≈+0.25/步`；一越过缺口近端、侧激光丢墙，**track 瞬间
  掉 0（奖励悬崖）**，桥接分支只剩 progress≈+0.23；
- 下一步策略立刻满幅倒车（动作 (-1.4,-1.3)、fwd=-0.7m/s）退回墙边重新拿回
  激光回波 —— 倒车段 `track(+0.22)` 与倒车 `progress(-0.22)` **几乎相抵、
  净亏≈0**；
- "前进丢墙 → 倒车找回 → 再前进"每 ~24 步一个周期，平均收益反超直行桥接
  （往复 ~0.35/步 > 桥接 ~0.26/步），策略稳定收敛在极限环上。

**内角**：9 回合抵达内角（front+side 同时命中），**9/9 全部停死**
（4.9~16.2s，多数到回合耗尽）。停死状态 fwd≈0、yaw_rate≈0（完全没在
转向）、动作≈0，每步 `track +0.10~0.28 + upright +0.05`、**零支出**
（wobble 被 front.hit 门控关掉、approach 不动不罚、progress 驻留为 0）
—— "停在角落收租"是纯正收益的局部最优，策略毫无转向动机。

共同根因：**track（0.28 权重，最大单项）对"不干活"的状态照常发放** ——
驻留白拿、倒车找回激光回波也白拿；而缺口桥接分支恰恰不发 track（悬崖）。
三个漏洞组合出上述两个行为缺陷。

### 改动内容（`tasks/wall_follow_task.py`）

1. **桥接虚拟贴边分**：`_gap_bridging` 命中时顺带算出机身到"原墙内侧面
   延长线"的贴边偏差（真值几何，目标=底盘半径+1cm），存 `bridge_lat_err`；
   `calc_reward` 桥接分支按贴墙**同一公式**发 track（exp 整形、8mm 尺度），
   progress 门控也改为与贴墙同式（`0.25+0.75*track_raw`）——
   **track 与 progress 跨缺口完全连续，"直行穿越"与"贴墙直行"等值**，
   且拐进缺口内侧（偏离墙线）拿不满分；
2. **倒车不发贴边分**：`rev_gate = clip((fwd+0.06)/0.06, 0, 1)` —— fwd≥0
   全额、≤-0.06m/s 全扣。掐掉往复循环唯一的收入来源，墙端/外角/U 型包边
   处的同构循环一并封死；
3. **驻留不发贴边分（运动门控）**：`move_gate = max(clip(fwd/(0.5·V_DES)),
   clip(|yaw_rate|/0.5))`，`track = track_raw × move_gate × rev_gate` ——
   正常巡航（≥0.14m/s）与**内角原地转向（≥0.5rad/s）都拿满**，停死拿 0，
   "转向离开角落"成为内角处唯一的高收益动作；直墙上"原地摇摆刷分"仍被
   贴墙 wobble 罚（-0.25×track/步）压住，不开新洞。

progress 门控与 wobble 门控仍用未扣发的 `track_raw`：倒车/驻留时贴边质量
本身没变差，速度罚不能因门控变小而被稀释。新增常量：`CHASSIS_RADIUS=0.175`
（与 gen_xml 一致）、`REV_TRACK_SCALE=0.06`。

### 验证

1. **状态注入数值检查**（真实 env 传送+注速/注转，全部断言过）：
   - 缺口：贴墙直行 +0.428/步 vs 缺口中点直行桥接 **+0.523/步**（旧版桥接

线性到 1）。问题：

- 它罚的是"近墙处的**全部**前向速度"，连"慢速小心接近"也一并罚 —— 尺度含糊，
  策略学到的是"笼统地慢一点"，而不是"到 1cm 处速度恰好归零"；
- 权重 0.18 偏弱、起罚距离 8cm 偏晚；蹭墙要**持续 1.5s** 才终止，轻轻碰一下
  几乎零代价 —— 于是策略容忍"碰一下再沿墙"，而不是"零接触停住"。

### 修复：把 approach 写成"由前向距离决定的前向限速"

`v_cap(d) = V_DES × clip((d − FRONT_STOP_GAP)/(FRONT_SAFE_DIST − FRONT_STOP_GAP), 0, 1)`
—— 正前方障碍在 **1cm(FRONT_STOP_GAP)** 处允许前向速度 0、在 **9cm(FRONT_SAFE_DIST)**
及以外允许满速 V_DES，中间线性。只惩罚"超过限速的部分"：
`approach = confidence × max(fwd_vel − v_cap, 0)/V_DES`。

- **直接编码"距墙 1cm 停住"**：梯度指向"贴到 1cm 前把速度降到 0"，而不是笼统减速；
- **不误伤**慢速接近 / 内角减速（都在限速内，罚 0），唯有"冲着墙加速"才吃罚；
- 置信度加权：矮墙只挡部分射线、看不真切时罚得轻；
- 权重 0.18 → **0.22**（增强刹车权威）；
- **head_on 课程加强**：占找墙回合比例 45% → 50%；`d0` 改为 **8~35cm**
  （8~10cm 出生的回合一睁眼就看到墙、必须立刻刹车 —— 最强刹车梯度；更远的
  先巡航、进 10cm 量程再刹）；出生偏航放宽到 +90°±(−0.6,+0.5)。

### 涉及文件

| 文件 | 改动 |
|---|---|
| `tasks/wall_follow_task.py` | 新增 `FRONT_STOP_GAP`、`FRONT_SAFE_DIST` 0.08→0.09；`approach` 重写为限速式 overspeed，权重 0.18→0.22；head_on 课程加强 |

### 验证

   1cm 还在动→重罚、1cm 已停→0；3cm 慢速 0.02→0（限速内）。逐条符合预期。
2. **真实 env 对照**（同一 head_on 出生点朝墙前进）：全力直冲 → 最小前向距离

3. **端到端**：2 迭代训练冒烟通过（回报 15.2→9.95，KL 0.005~0.015 健康）。

## [2026-07-09] 墙体缺口（门洞/断墙）桥接：过缺口不再丢墙重搜

### 背景 / 现象

沿边行进时，前方出现一段**两端有墙、中间无墙、地面贯通**的缺口（门洞/断墙）。
机器人抵达缺口、左侧激光丢墙那一刻，策略把它误当"外角/彻底丢墙"处理：
中途减速、原地转向、重新找墙，**连续沿边作业被打断**，不再沿原墙轮廓续行。

### 根因：训练课程里从来没有"缺口"，奖励把"缺口"和"丢墙"混为一谈

- 课程 `_build_random_course` 只会摆**连续折线墙**（段与段之间必是拐角），
  从没出现过"墙—缺口—共线续墙"，策略没学过"直行桥接"这一行为；
- 奖励在丢墙后只有两条分支：`lost_time<1.2s` 的**外角包边宽限**（直行）、
  之后的**真找墙**（`progress*=0.2` 弱化前进 + 吃满丢墙罚）。缺口只要比
  `V_DES×1.2s≈18cm` 宽，跨越途中就落进"真找墙"分支 —— 奖励**主动**把机器人
  从"匀速直行"推向"减速+转向去重新找墙"，这正是观察到的坏行为的直接来源。

### 修复：课程加缺口 + 奖励用真值区分"缺口 vs 外角"

**1) 课程（`_build_random_course`）**：热身后（frac>0.15）在**共线段边界**以
`GAP_PROB=0.35` 概率插入缺口，保持墙向不变、跨过 `GAP_MIN~gap_max` 的无墙段
再续共线墙；`gap_max` 随课程从 0.20m 升到 0.55m。缺口两端墙共线（续墙不做
拐角处的 10cm 回收，避免吃掉缺口），真值记进 `self.gaps=[(近端墙角, 墙向, 宽)]`。

**2) 奖励（`calc_reward`）新增缺口桥接分支**（在外角宽限之前）：用真值
`_gap_bridging()` 判断"是否正沿缺口直行、且前方有共线墙可续贴"：
- 满足 → `progress*=1.0`（全速直行桥接）、`lost*=0.15`（几乎不罚丢墙），
  并把 `wobble` 门控开满（`steady=1`）压住偏航 —— 逼机器人**直行穿过**、
  别拐进缺口；到对侧 `left.hit` 自然回到贴墙分支；
- 不满足（外角处前方**没有**共线墙）→ 走原来的外角包边逻辑，**不受影响**。

**3) 终止（`done`）**：正在桥接缺口时不按"丢墙超时"终止（宽缺口跨越耗时可能
逼近 `LOST_TERM_TIME=4s`），避免中途误杀 episode。

### 关键取舍：缺口宽度上限为何保守（0.55m）

策略是**纯反应式**（只有 2 个短程激光 + 0.2s 历史）。在丢墙那一刻，"缺口"与
"外角"的激光观测**完全一致**（都是左侧空、前方畅通），二者仅在"继续直行后墙
是否在对侧重现"上不同——这是个时序差异，反应式策略无法瞬时区分。因此奖励用
真值区分只是**把正确动作教给策略**，策略最终学到的是"丢墙后直行 D* 距离再考虑
左勾"的折中，D* 由课程里缺口/外角的比例与宽度分布决定。缺口越宽，就越要求策略
直行更久，代价是外角处越容易冲过头。上限取 0.55m 是为了**桥接中小缺口、又不
明显回退昨天刚修好的外角包边（阳角卡角）**。>0.55m 的宽门洞暂列为已知局限。

### 涉及文件

| 文件 | 改动 |
|---|---|
| `tasks/wall_follow_task.py` | 新增缺口常量 / `self.gaps`/`self.bridging`；`_gap_bridging()` 真值判定；`step()` 每步缓存 `bridging`；`calc_reward` 缺口桥接分支 + `wobble` 门控扩展；`done` 桥接时不按丢墙终止；`_build_random_course` 插入共线缺口 |

### 验证

1. **`_gap_bridging` 纯逻辑**（8 个用例全过）：缺口内朝墙向直行→True；
   缺口内朝墙外拐 90°/横向跑偏 0.5m/未到/已过/反向→False；空 gaps→False。
2. **课程真的摆出缺口**：40 次 reset（frac=1.0）中 12 次生成缺口，两端墙共线
   （墙向一致）、宽度落在课程区间。
3. **奖励分支生效**：把机器人放进缺口中点朝墙向→`bridging=True`，
   `reward.lost=-0.0075`（桥接分支 = -0.05×0.15），非桥接丢墙为 -0.05。
4. **环境无异常**：300 步随机动作 step/reward/done 跑通，reward 均有限。
5. **端到端**：2 迭代训练冒烟通过（4 进程并发、回报 8.7、KL≈0.007 健康）。

### 兼容性影响

- **观测/动作维度不变（34/2），奖励 dict 键不变** —— PPO / 回放脚本无需改动；
- 奖励函数已改，**需重训**才能让策略学会桥接缺口（旧模型仍可加载回放，
  但不会桥接）；家居场景回放 `gaps=[]`，`bridging` 恒 False，回放行为由重训后
  的策略决定。
- 规则控制器 `wall_follower.py` **未改**：其 `WRAP_LEFT` 在宽缺口处同样会误当
  外角左勾（已知局限，本次未涉及）。

## [2026-07-08] 重训后三个行为缺陷修复：阳角卡角 / 沿墙摇晃 / 家居场景撞墙

### 背景 / 动机

幽灵墙修复后重训的模型，仿真中暴露三个行为缺陷：

1. **阳角（外角）转弯时墙角卡进机身**；
2. **沿墙行进时机身一直摇晃**；
3. **家居场景里先直冲撞墙、再开始正常沿墙**。

逐个定位后发现：#2 的主因是一个**一直存在的代码 bug**（平滑惩罚恒为 0），
#1 和 #3 则是奖励/课程的设计缺口（分别缺"包边宽限"和"前向减速"）。

### 根因 1（bug）：平滑惩罚 `smooth` 恒等于 0 —— 沿墙摇晃

- `envs/vacuum/robot.py` 的 `step()` 在 `do_simulation` 之后把 `prev_action`
  覆盖成**本步**动作；而 `env.step()` 的顺序是先 `robot.step(a)` 再
  `calc_reward(..., self.robot.prev_action, a)`。到达奖励函数时
  `prev_action` 已经等于 `a`，于是 `dact = a - a ≡ 0`。
- **后果**：`smooth` 项自始至终是 0，动作变化率从未被惩罚 —— 策略可以任意
  高频抖动而零代价，机身自然一直摇晃。`vacuum` 过坎任务走同一条代码路径，
  同样中招。
- **修复**：`VacuumRobot` 新增 `last_action / last_torque`，在覆盖 `prev_*`
  之前抓取真正的上一步值；两个 env 的 `calc_reward` 改传 `last_action`。
  `prev_torque` 保持不变传给能耗项（它本就该是"本步实际施加的力矩"）。
  观测里的 `prev_action` 也保持不变（观测在 `robot.step` 之后取，
  "最近一次动作"正是策略该看到的量）。

### 根因 2：奖励里没有"接近障碍必须减速"项 —— 家居场景撞墙

- 前向激光虽然在观测里（34 维中占 2 维 + 历史帧），但**奖励从不依赖它**，
  策略没有任何理由学会用它刹车。训练里墙又永远在左侧、从不正面出现，
  于是"正对墙面"这个状态在训练分布里几乎不存在（OOD）。
- **修复（两处，缺一不可）**：
  - 奖励新增 `approach` 项：正前方 8cm 内有障碍时，
    `罚 ∝ 置信度 × 接近程度 × max(前向速度, 0)` —— 只在真的朝障碍前进时生效；
  - 课程新增 **head_on 出生模式**（占找墙回合的 45%）：机器人正对墙面
    （yaw≈+90°，离墙 12~30cm）出生，必须减速并转向对齐才能沿墙。

### 根因 3：阳角处策略"急着左勾"把墙角勾进机身

- 左侧激光装在机体**横向中心线**（x=0）上。丢墙那一刻，墙角恰好在机身正侧方；
  此时若立刻左转，机身左前缘会扫向仍在原地的墙角 → 卡角。
- 旧奖励在丢墙时 `progress *= 0.2` 且吃满 `lost` 罚，等于**逼**策略尽快左勾找墙。
- **修复**：识别阳角信号（左墙刚丢 `lost_time < 1.2s` **且**前方无障碍），
  在该窗口内 `progress *= 0.6`、`lost *= 0.3` —— 鼓励先直行越过墙角再左转。
  窗口 1.2s 由几何定出：把墙角甩到机身后方需再前进约一个底盘半径
  （`0.175m / V_DES 0.15 ≈ 1.17s`）。
  该策略与规则控制器 `WRAP_LEFT`（先直行 20cm 再画圆弧）的做法一致。
- 退化解检查：窗口内"贴着量程边缘吊着走"最多拿 `0.26×0.6 − 0.05×0.3 ≈ 0.14/步`，
  远低于正常贴墙跟踪（`track+progress` 可达 ~0.45/步），不构成新的局部最优。

### 关键取舍：抖动抑制项该罚什么？（实测否定了第一版设计）

先试了罚**偏航角加速度**（"正常拐弯是恒角速度圆弧、摇晃是角速度反复反号"）。
实测直接否掉：规则控制器的 `|yaw_accel|` 均值已达 **8.6 rad/s²**（被接触/求解
噪声淹没），注入振荡后只涨到 9.7（**1.1x，无判别力**）；`rms|dact|` 同样只有
1.0x。**唯一有判别力的是直行门控下的 `|yaw_rate|`**（注入振荡 1.5~2.0x，且随
振幅单调）。原设计的信号是对的，错在尺度：`(yaw_rate/2.0)²` 让规则控制器只被
罚 -0.0006，比 track 收益小 200 倍，等于没有。

于是在实测分布上标定 `wobble = track × min((|yaw_rate| / S)², 1)`：

| S | rule | wob0.3 | wob0.5 | wob0.8 | wob0.8/rule |
|---|---|---|---|---|---|
| 0.4 | -0.0147 | -0.0193 | -0.0244 | -0.0294 | 2.00x |
| **0.6** | **-0.0080** | **-0.0112** | **-0.0157** | **-0.0222** | **2.78x** |
| 0.8 | -0.0048 | -0.0067 | -0.0099 | -0.0155 | 3.20x |
| 2.0（原） | -0.0010 | -0.0012 | -0.0017 | -0.0028 | 2.90x |

取 **S=0.6, W=0.10**：判别比 2.78x，绝对量级够形成梯度，规则控制器只被罚掉
track 收益的 7%；且远低于丢墙罚 0.05/步 —— 策略不会为躲这项而放弃贴墙
（`wobble` 被 `track` 门控，不贴墙反而损失更大）。

### 奖励权重调整

## [2026-07-07] 沿边 RL 训练失败根因修复：幽灵墙 / 奖励漏洞 / 泛化缺陷

### 背景

`vacuum_wf` 首轮 1500 迭代训练后效果极差：随机墙场景中机器人撞角后**整体嵌入墙体**；
家居泛化场景中**原地打转**，完全没有沿边行为。排查后定位到一个物理层 bug（主因）
和多个奖励/课程设计缺陷（次因）。

### 根因 1（主因）：运行时摆放的墙是"幽灵"，物理上可穿透

- **现象**：最小隔离实验证实，MuJoCo 2.3.6 在**编译期**把世界静态 geom 的
  位置/尺寸烘焙进碰撞加速结构，运行时修改 `model.geom_pos / geom_size`
  **对碰撞检测完全不生效**（激光 `mj_ray` 却读实时位置，"看得见、撞不着"）。
- **后果**：训练时所有随机墙没有物理实体，策略学到"穿墙走"；到家居场景
  （XML 原生墙、碰撞真实）立即崩溃。**原 `vacuum` 过坎任务的门槛（box01 运行时
  摆放）也是同样的幽灵，此前该任务的物理同样是坏的。**
- **修复**：所有动态地形改为 **mocap body**（MuJoCo 运行时摆放物体的标准机制，
  碰撞跟随 `data.mocap_pos/quat`）。约束：geom 尺寸必须固定 → 采用固定尺寸组合：
  - 沿边训练墙 `box01~06`：半长 0.80/0.60/0.45 各两块，半厚 0.08、半高 0.125；
  - 过坎门槛 `ridge01`(0.06,1.00,0.05)、AGAINST 短墙 `ridge02`(0.06,0.30,0.05)、
    骑坎窄脊 `ridge03`(1.00,0.05,0.05)；
  - **不同有效墙高用"下沉入地"实现**（mocap body 与 world 同属一个 weld，
    与地面不产生接触，可以任意下沉）。

### 根因 2：奖励函数存在"压墙滑行"局部最优

- **现象**：斜压在墙上滑行可白拿满速度分（progress 只看前向速度），蹭墙惩罚
  0.10 太轻，净收益为正 → 策略收敛到贴墙硬推。
- **修复**（`tasks/wall_follow_task.py`）：
  - progress 用贴边质量门控：`progress *= (0.25 + 0.75*track)`，贴不准拿不到速度分；
  - 蹭墙惩罚 0.10 → 0.20，且**机身持续接触 1.5s 直接终止 episode**；
  - 修复后实测：压墙滑行 = **-0.119/步且 75 步被终止**（修复前 +0.075 跑满全程）。

### 根因 3：策略从没学过"找墙" → 家居场景出生点观测出分布(OOD)

- **修复**：35% 回合出生在离墙 12~45cm 处（找墙课程），首次捕获墙面前丢墙宽限
  8s（捕获后 4s）。策略必须学会"先接近、捕获墙面，再贴边"。

### 根因 4：拐角处部分可观测，前馈网络单帧激光分不清"刚丢墙"与"一直没墙"

- **修复**：观测追加激光历史帧（前向/左侧的距离+置信度在 0.05/0.1/0.2s 前的
  快照，共 12 维），**观测 22 → 34 维**。

### 附带修复：HIGHCENTER（骑坎）场景几何错误

- 原设计凸脊沿 y 摆放，实际会先托住中线上的前后万向球 → 轮子完全悬空，
  是不可自救的死局（此前被幽灵物理掩盖，从未真实生效）。
- 改为 `ridge03` 窄脊（宽 10cm）沿行进方向、横向偏置 ±8cm，从万向球
  (|y|<0.022) 与驱动轮 (|y|>0.138) 之间的空隙下方穿过：腹部压脊、机身微倾、
  单侧轮打滑但另一侧仍有抓地 —— 可脱困，符合"骑坎打滑"的设计意图。

### 其他加固

- 训练墙/门槛接触加硬（`geom_solref=(0.005,1)`, `solimp=(0.95,0.99,0.001)`），
  防止大力矩顶墙深度穿透（`envs/vacuum/vacuum_wf_env.py`）。

### 涉及文件

| 文件 | 改动 |
|---|---|
| `envs/vacuum/gen_xml.py` | box 由静态 geom 改为 mocap body；新增 `WALL_BOX_SPECS`/`RIDGE_BOX_SPECS` 固定尺寸组合 |
| `tasks/wall_follow_task.py` | mocap 摆墙 + 下沉调高；奖励门控与蹭墙终止；找墙课程；`_bury_all_boxes` 遍历 mocap body |
| `tasks/vacuum_task.py` | 门槛/短墙/骑坎脊改用 mocap（ridge01/02/03）；HIGHCENTER 几何重设计 |
| `envs/vacuum/vacuum_wf_env.py` | 激光历史帧观测(34 维)；地形接触加硬 |
| `scripts/test_line_laser.py` | 适配 mocap 摆放；7cm 边界用例预期修正、新增 8cm 用例 |

### 验证

1. 最小隔离实验：静态 geom 运行时改 pos/size → 无碰撞；mocap body → 碰撞正确
   （含旋转、下沉、埋地、动力学承重）；
2. 奖励排序：规则控制器 +0.456/步 ≫ 压墙 -0.119（被终止）≫ 打转 +0.001；
3. 全力矩正面顶墙 8s：机身被真实挡住（修复前直接穿墙飞出 6m）；
4. 激光特性 11 项断言全过；CROSS 门槛/HIGHCENTER 骑脊接触真实；
5. 12 迭代试训练：评估回报 51.6 → 86.7 稳步上升，KL≈0.006 健康。

### 兼容性影响（重要）

- **观测维度 22→34：`vacuum_wf` 旧模型全部作废，必须从头重训**；
- `vacuum` 过坎任务此前的物理是坏的，**旧的过坎训练结果无效，需要重训**；
- 规则控制器 demo（`wall_follow_demo.py`）不受影响（家居墙为 XML 原生）。

## [2026-07-07] PPO 接入沿边任务 + 算法优化 + 边刷旋转

### 新增

- **`vacuum_wf` 环境**（`envs/vacuum/vacuum_wf_env.py`，注册于 `run_experiment.py`）：
  沿边沿墙 RL 训练环境。观测只含激光 + 本体感知（无全局坐标/特权信息），
  这是策略能泛化到任意户型的关键；`home_scene=True` 变体用于泛化回放。
- **`WallFollowTask`**（`tasks/wall_follow_task.py`）：随机折线墙课程（段数/拐角
  角度方向/墙高随迭代展开）、激光噪声课程(0→2mm)、地面摩擦随机化；
  奖励 = 贴边(1cm) + 沿墙推进 + 水平 − 丢墙 − 蹭墙 − 能耗 − 平滑。
- **回放脚本** `scripts/debug_wallfollow.py`（`--home` 家居泛化验证）。

### PPO 算法优化（`rl/algos/ppo.py`）

- **实现真 GAE(λ)**：原 `use_gae` 参数形同虚设（优势=MC回报−V，方差大）。
  现采样时反向扫描计算 GAE 优势，critic 目标改用 TD(λ) 回报；
- **KL 早停可用**：`target_kl` 原写死 None，现由 `--target_kl` 传入（建议 0.04）；
- **`--lr_decay`**：学习率随迭代线性衰减（至 5% 下限）。
- 均向后兼容：不传新参数时行为与原版一致。

### 边刷旋转（`envs/vacuum/gen_xml.py` + `envs/vacuum/robot.py`）

- 左右三叶边刷改为挂在底盘上的独立 body + 绕 z 铰链关节，`VacuumRobot`
  每个仿真子步以 12 rad/s 运动学驱动（左刷顺时针/右刷逆时针，向机身中线扫拢）。
  不占执行器，不改动作/观测维度；旧 `vacuum` 环境观测仍为 23 维。

### 验证

- 边刷转向/转速正确；`vacuum` 环境兼容冒烟通过；
- 3 迭代端到端训练冒烟通过（ray 12 进程 XML 按 pid 隔离写入，无并发冲突）。

## [2026-07-06] 沿边沿墙功能落地：线激光建模 + 规则沿墙控制器

### 新增

- **线激光传感器 MuJoCo 建模**（`envs/vacuum/gen_xml.py`）：
  前向/左侧各一个，每个用同一位置、离地 3.5/4.7/5.8/7.0cm 的 4 条 rangefinder
  射线近似竖直激光条纹；`cutoff=0.10` 体现 10cm 量程。特性由几何自然呈现：
  ≥7cm 全命中（可靠）、4~6cm 部分命中（低置信）、2~3cm 低于最低射线（盲区）。
- **传感器读取封装**（`envs/vacuum/sensors.py`）：`LineLaser.read()` →
  `distance / confidence / hit`，支持测距噪声；注意 cutoff 会把超量程命中钳到
  恰好 0.10，判定必须用严格小于。
- **规则沿墙控制器**（`envs/vacuum/wall_follower.py`）：
  状态机（SEEK 找墙 / TURN_RIGHT 内角 / FOLLOW 贴边 / WRAP_LEFT 外角包边 /
  ESCAPE 盲区脱困）+ 横向距离 PD（KP=8/KD=16，按闭环 `e''+vKD·e'+vKP·e=0`
  临界阻尼整定）+ 差速运动学 + 轮速 PI → 轮力矩。
  关键细节：右转对齐以"左侧距离越过最小值回升"判定；盲区障碍（毯沿等激光
  看不见）用里程计失速检测触发倒车换向脱困。
- **演示/验证脚本**：`scripts/wall_follow_demo.py`（激光射线可视化 + 贴边统计）、
  `scripts/test_line_laser.py`（量程/检测高度特性断言）。

### 场景调整

- 地毯从北墙沿边路径挪到外间东南空地并降为 12mm：原 16mm > 15mm 底盘间隙
  且在激光盲区，正好压在沿边环线上会把底盘卡死；
- 环境暴露 `env.laser_front / env.laser_left`，RL 观测保持 23 维不变。

### 验证

- 传感器特性断言全过（测距 3/6/9cm 准确、12cm 无回波、2/3cm 盲区、5cm 低置信、
  ≥7cm 可靠）；
- 300s 无渲染仿真：完整巡边环线可重复（北墙→沙发轮廓→隔断→门洞鼻端包边→
  南墙→木柜轮廓→西墙→闭环），稳态贴边误差中位 1.4mm、直线段稳定 10mm，
  盲区障碍（充电座底板）触发脱困后正常继续。
